## Risks & Blockers

### 🔴 High Priority
1. **Booking route guard (`BookingRouteGuard`)** — uses React Router state (location state) to enforce funnel order. Must be re-implemented in Next.js using cookies or URL params since `useLocation()` doesn't exist.
2. **GHL API proxy (`/api/ghl`)** — client fetches use `window.location` for redirects. Port to Next.js API routes before any booking flow migration.
3. **Analytics guard** — sanitizes booking-route URLs before GTM fires (HIPAA/PHI concern). Must be ported before booking flow goes live in Next.js — cannot skip.

### 🟡 Medium Priority
4. **Zustand store shape** — booking state stored in localStorage-backed Zustand. Needs audit before migrating booking flow.
5. **`motion` (Framer Motion v12) API** — v12 has breaking changes from v10/v11. Verify current usage before upgrading.
6. **Oswald + Inter fonts** — currently loaded via `index.html` Google Fonts links. Port to `next/font/google` for self-hosting.
7. **Brand tokens CSS** (`brand-tokens.css`, `cro-tokens.css`) — uses CSS custom properties. Should import in `globals.css` — straightforward port.

### 🟢 Low Risk
8. **shadcn/ui components** — all Radix-based, work fine in Next.js with `'use client'` directive
9. **Tailwind config** — v3 config, compatible with Next.js setup
10. **TypeScript** — strict mode already used in mwc-v2
11. **Environment variables** — Vite uses `VITE_*` prefix. Next.js uses `NEXT_PUBLIC_*` for client-exposed vars. All env vars need renaming.
